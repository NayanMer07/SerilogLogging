Why structured logging matters in production .NET apps

❌ Traditional string logging

Queryable · filterable · alertable

Serilog plugs into the standard Microsoft.Extensions.Logging.ILogger — no refactoring your existing controller code.

Industry-Standard Serilog Code

Program.cs bootstrap, enrichers, request logging, and structured log calls

🚀Program.cs — Bootstrap Serilog Early

C# · Program.cs
using Serilog;

// ── Bootstrap logger (captures startup errors before config loads) ─
Log.Logger = new LoggerConfiguration()
    .WriteTo.Console()
    .CreateBootstrapLogger();

try
{
    Log.Information("Starting OrderApi");

    var builder = WebApplication.CreateBuilder(args);

    // ── Replace default logger with Serilog ────────────────────────
    builder.Host.UseSerilog((ctx, services, config) => config
        .ReadFrom.Configuration(ctx.Configuration)   // reads appsettings.json
        .ReadFrom.Services(services)                  // allows DI in sinks
        .Enrich.FromLogContext()                      // LogContext.PushProperty
        .Enrich.WithMachineName()
        .Enrich.WithThreadId()
        .WriteTo.ApplicationInsights(
            services.GetRequiredService<TelemetryConfiguration>(),
            TelemetryConverter.Traces));

    builder.Services.AddControllers();
    builder.Services.AddApplicationInsightsTelemetry();
    builder.Services.AddScoped<IOrderService, OrderService>();

    var app = builder.Build();

    // ── Serilog HTTP request logging (replaces default middleware) ──
    app.UseSerilogRequestLogging(options =>
    {
        options.MessageTemplate =
            "HTTP {RequestMethod} {RequestPath} responded {StatusCode} in {Elapsed:0.0000} ms";
        options.EnrichDiagnosticContext = (diag, http) =>
        {
            diag.Set("RequestHost", http.Request.Host.Value);
            diag.Set("UserAgent",   http.Request.Headers["User-Agent"]);
        };
    });

    app.MapControllers();
    app.Run();
}
catch (Exception ex) when (ex is not HostAbortedException)
{
    Log.Fatal(ex, "OrderApi terminated unexpectedly");
}
finally
{
    Log.CloseAndFlush();  // always flush before exit
}

📦OrderService.cs — Structured Log Calls

C# · OrderService.cs
namespace OrderApi.Services;

public record Order(int Id, string CustomerEmail, decimal Total, string Status);

public class OrderService : IOrderService
{
    private readonly ILogger<OrderService> _logger;

    public OrderService(ILogger<OrderService> logger) => _logger = logger;

    public async Task<Order> PlaceOrderAsync(Order order)
    {
        // ✅ Structured: OrderId and CustomerEmail are queryable properties
        _logger.LogInformation(
            "Placing order {OrderId} for {CustomerEmail} total {Total:C}",
            order.Id, order.CustomerEmail, order.Total);

        try
        {
            // simulate DB save
            await Task.Delay(50);

            // Push additional context for all logs in this scope
            using var scope = _logger.BeginScope(new Dictionary<string, object>
            {
                ["OrderId"]       = order.Id,
                ["CustomerEmail"] = order.CustomerEmail
            });

            var confirmed = order with { Status = "Confirmed" };
            _logger.LogInformation(
                "Order {OrderId} confirmed successfully", confirmed.Id);

            return confirmed;
        }
        catch (Exception ex)
        {
            // ✅ Exception is captured as a structured property — not just a string
            _logger.LogError(ex,
                "Failed to place order {OrderId} for {CustomerEmail}",
                order.Id, order.CustomerEmail);
            throw;
        }
    }

    public async Task CancelOrderAsync(int orderId, string reason)
    {
        _logger.LogWarning(
            "Order {OrderId} cancelled — Reason: {CancellationReason}",
            orderId, reason);

        await Task.Delay(20);
    }
}

🌐OrdersController.cs — Request Context Enrichment

C# · OrdersController.cs
[ApiController]
[Route("api/[controller]")]
public class OrdersController : ControllerBase
{
    private readonly IOrderService _orders;
    private readonly ILogger<OrdersController> _logger;

    public OrdersController(IOrderService orders,
                             ILogger<OrdersController> logger)
        => (_orders, _logger) = (orders, logger);

    [HttpPost]
    public async Task<IActionResult> PlaceOrder([FromBody] Order order)
    {
        // Enrich ALL logs in this request with CorrelationId
        using (LogContext.PushProperty("CorrelationId",
               HttpContext.TraceIdentifier))
        {
            _logger.LogInformation(
                "POST /orders received for {CustomerEmail}",
                order.CustomerEmail);

            var confirmed = await _orders.PlaceOrderAsync(order);
            return CreatedAtAction(nameof(GetOrder),
                   new { id = confirmed.Id }, confirmed);
        }
    }

    [HttpGet("{id}")]
    public IActionResult GetOrder(int id)
    {
        _logger.LogDebug("Fetching order {OrderId}", id);
        // ... return order
        return Ok(new { id });
    }
}
💡Key rule: Always use message templates with named properties like "Order {OrderId} placed" — never string interpolation like $"Order {id} placed". Templates are the heart of structured logging — they preserve the property separately so you can filter by it.

Sinks & Log Levels

Where logs go and how to control the noise

🖥️
Console
Serilog.Sinks.Console
Human-readable output during development. Coloured by log level. Zero config needed.
📁
File
Serilog.Sinks.File
Rolling daily log files. Auto-deletes old files. JSON or plain text output. Essential for on-premise.
🔍
Seq
Serilog.Sinks.Seq
Free local structured log viewer. Query logs like a database. Best dev experience for Serilog.
☁️
Application Insights
Serilog.Sinks.ApplicationInsights
Azure's cloud telemetry platform. Live metrics, alerts, distributed tracing. Production standard.
📧
Email
Serilog.Sinks.Email
Send FATAL errors to your inbox immediately. Pair with minimum level filter for fatal-only alerts.
🗄️
MS SQL Server
Serilog.Sinks.MSSqlServer
Write structured logs to a SQL table. Queryable with standard SQL. Good for enterprise audit trails.

How to Utilize Serilog

Advanced patterns — scopes, custom enrichers, audit logs, and alerts

🔗Correlation IDs — Trace a Request Across Services

C# · Correlation Middleware
public class CorrelationMiddleware(RequestDelegate next)
{
    public async Task InvokeAsync(HttpContext ctx)
    {
        // Read from incoming header or generate new ID
        var correlationId = ctx.Request.Headers["X-Correlation-ID"]
            .FirstOrDefault() ?? Guid.NewGuid().ToString("N");

        // Push to ALL logs in this request scope
        using (LogContext.PushProperty("CorrelationId", correlationId))
        using (LogContext.PushProperty("UserId",
                ctx.User?.Identity?.Name ?? "anonymous"))
        {
            ctx.Response.Headers["X-Correlation-ID"] = correlationId;
            await next(ctx);
        }
    }
}

// Register in Program.cs
app.UseMiddleware<CorrelationMiddleware>();

📋Audit Logging — Separate Sink for Compliance

C# · Audit Logger
// Create a dedicated audit logger separate from app logger
var auditLogger = new LoggerConfiguration()
    .MinimumLevel.Information()
    .WriteTo.File(new JsonFormatter(), "logs/audit-.json",
        rollingInterval: RollingInterval.Day,
        retainedFileCountLimit: 365)  // keep 1 year
    .Enrich.FromLogContext()
    .CreateLogger();

// Register in DI with a named logger
builder.Services.AddKeyedSingleton<ILogger>("Audit", auditLogger);

// Use in a service
using (LogContext.PushProperty("AuditAction", "OrderPlaced"))
using (LogContext.PushProperty("UserId", userId))
{
    _auditLogger.Information(
        "Order {OrderId} placed by {UserId} for {Total:C}",
        order.Id, userId, order.Total);
}

KUSTO · Query Logs in Azure
// In Azure Monitor → Logs (Kusto Query Language)

// Find all error logs with structured property
traces
| where severityLevel >= 3   // 3 = Error, 4 = Fatal
| where customDimensions.OrderId != ""
| project timestamp, message, customDimensions.OrderId,
          customDimensions.CustomerEmail, operation_Id
| order by timestamp desc

// Alert: error rate > 10 in 5 minutes → page on-call
traces
| where severityLevel == 3
| summarize ErrorCount = count() by bin(timestamp, 5m)
| where ErrorCount > 10

// Trace a request by CorrelationId across all log events
traces
| where customDimensions.CorrelationId == "abc123def456"
| order by timestamp asc

⚡Custom Enricher — Add User Context Automatically

C# · Custom ILogEventEnricher
public class UserContextEnricher(IHttpContextAccessor accessor)
    : ILogEventEnricher
{
    public void Enrich(LogEvent logEvent, ILogEventPropertyFactory factory)
    {
        var user = accessor?.HttpContext?.User;
        if (user?.Identity?.IsAuthenticated == true)
        {
            logEvent.AddPropertyIfAbsent(
                factory.CreateProperty("UserId",
                    user.FindFirst("sub")?.Value));
            logEvent.AddPropertyIfAbsent(
                factory.CreateProperty("UserEmail",
                    user.FindFirst("email")?.Value));
        }
    }
}

// Register in Program.cs
builder.Services.AddHttpContextAccessor();
// then in UseSerilog():
.Enrich.With(new UserContextEnricher(
    app.Services.GetRequiredService<IHttpContextAccessor>()))

Best Practices

What separates amateur logging from production-grade observability

✍️Use Message Templates

Always write _logger.LogInformation("Order {OrderId}", id) — never $"Order {id}". String interpolation destroys structure and makes logs unsearchable.

🚀Bootstrap Logger First

Configure the bootstrap logger at the very start of Program.cs before anything else. This captures startup crashes that occur before the DI container is built.

🔇Suppress Framework Noise

Override Microsoft and System to Warning in config. Otherwise 90% of your logs are framework internals with no business value.

📤Always Flush on Exit

Call Log.CloseAndFlush() in the finally block. Serilog buffers events — without flushing, the last seconds of logs before a crash are lost.

Production Checklist

PracticeWhy it matters
✓Message templates for all structured propertiesEnables filtering, alerting, and grouping in any log sink
✓Bootstrap logger before DI container buildsCaptures fatal startup exceptions before sinks are ready
✓Log.CloseAndFlush() in finally blockGuarantees buffered events reach all sinks before process exit
✓Suppress Microsoft/System to WarningReduces log volume by 80–90% with zero information loss
✓Correlation ID on every requestTrace a single request across all microservices end-to-end
✓Different log levels per environmentDebug in dev, Information in staging, Warning in prod
✓Log connection string injected from env varsNo secrets or API keys hardcoded in appsettings.json
✓Rolling file with retention limitPrevents disk exhaustion — 7 days local, unlimited in cloud
✓Separate audit log sinkCompliance-grade immutable audit trail separate from app logs
